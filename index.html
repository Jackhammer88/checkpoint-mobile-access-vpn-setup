Fedora GNU/Linux setup Check Point Mobile Access VPN

This is a step-by-step tutorial to setup your Fedora GNU/Linux machine with all the required dependencies to work with Check Point Mobile Access VPN.

Introduction

If you have fedora 38 it must be hard to setup Check Point Mobile Access VPN on the fly. I created this guide so as not to forget how the setup is done myself, but if it will be useful to you, then I will be glad.

Dependencies

To setup this, you need to install this packages in Fedora 38.

sudo dnf install nss-tools openssl xterm
sudo dnf install glibc.i686 pam.i686 libX11.i686 libnsl.i686

You also must download compat-libstdc++-33.rpm package and install that.

sudo dnf install ./compat-libstdc++-33-3.2.3-72.el7.i686.rpm

You also have to install java.

curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"

After installing sdkman you should find the name of java sdk package. Choose the appropriate version (version 8) from the list.

sdk list java

... and install it.

sdk install java 8.0.232-open

After installing java do login to the Check Point Mobile Access VPN website. Try to connect. The website will offer you to download cshell_install.sh. Download it and install to your system.

bash ./cshell_install.sh

If cshell_install script installed successfully, the website will try to connect to the checkpoint vpn.
It is possible that you will have to manually stop the java process and restart the launcher.

ps -A | grep java
kill [your pid]
/bin/cshell/launcher

That's it!
